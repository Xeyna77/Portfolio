Le Concept

L'objectif principal de ce projet était de plonger dans l'écosystème Node.js à travers la création d'un bot Discord interactif. Ce projet m'a permis de comprendre comment JavaScript s'exécute côté serveur et comment gérer des flux de données bidirectionnels entre un client et une API.

Réalisation & Apprentissages

  • Maîtrise de l'Asynchronisme : Utilisation intensive des async/await et des Promises pour gérer les réponses de l'API Discord sans bloquer le thread principal.
  • Architecture Event-Driven : Compréhension du fonctionnement de l'Event Loop de Node.js via la gestion des événements (messages, réactions, connexions aux serveurs).
  • Intégration d'API : Connexion à des services tiers (météo, actualités ou jeux) pour enrichir les commandes du bot via des requêtes HTTP.
  • Sécurité des données : Gestion des variables d'environnement (fichiers .env) pour protéger les tokens d'accès sensibles.

Pourquoi Node.js ?

Ce projet a été le déclencheur de ma compréhension des systèmes non-bloquants. J'ai appris à structurer mon code de manière modulaire, en séparant les commandes dans des fichiers distincts pour rendre le projet scalable et facile à maintenir.
